Greendoor prototype

The greendoor prototype was part of the human-centred design course to answer the question: How might we help suburban homeowners realize the benefits of making low-carbon choices? The proposed idea included a digital energy management tool that helps homeowners shrink their carbon footprint and their energy bills. Low-fidelity prototype and app screens were developed and tested with users to gather feedback and iterate on design solutions.

  • My role: User research, user interviews, prototype creation
  • Completed: August 2017

MOECC's Intranet

  Spotlight Award winner for 'Internal Communications' (2014)

The Ministry of the Environment and Climate Change's intranet website is a tool utilized by over 2000 staff. The entire content management site was developed in-house using MVC methodology. Careful design considerations were made on how the platform would provide relevant content, promote engagement and enhance search.

  • My role: Visual design, UI design, information architecture, taxonomy development, front-end development
  • Tools utilized: HTML, CSS, PHP, Javascript, Photoshop
  • Completed: July 2012

MOECC's chatbot prototype

OPS first venture into chatbots, using MOECC's Drive Clean program as a sample topic. User design considerations included how the chat application would appear on various applications and platforms, the type of language used by the bot and how the user would interact through the work-flow process.

  • My role: User experience, knowledge base research, usability testing
  • Tools utilized: WIT.ai, PHP, Facebook
  • Completed: October 2017

CBTracker

Website

CBtracker is an internal web application for the Communications Branch at MOECC to track all unit projects and client requests. User research with staff involved with previous tracking methods revealed frustration with numerous documents and formats used. A centralized and simplified user interface was developed with features added as testing and feedback reported.

  • My role: User experience, user research, personas, information architecture, front-end development
  • Tools utilized: HTML, CSS, PHP, Javascript, Bootstrap
  • Completed: July 2012

Ezone

Application

E-Zone is a tablet application aimed at providing environmental tips for children. Illustrator artwork created by a vendor for other static E-Zone material was re-purposed to create a robust, interactive application. Interface considerations included how a young audience (4-9 year olds) would move through the various sections, resources and screens in the application. User testing involved teachers introducing E-Zone as a learning tool in their classrooms.

  • My role: User experience, user flows, graphic development, animation
  • Tools utilized: Illustrator, Photoshop
  • Completed: April 2014
